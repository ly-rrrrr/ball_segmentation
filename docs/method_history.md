# Method History

## 1. SAM 候选生成

项目首先使用 Transformers 的 SAM 自动掩膜 pipeline。为提高显微图中小球召回率，引入规则网格切分和 tile 放大后推理。该策略能产生更细的候选，但也带来重复掩膜、tile 边界碎片和大量背景伪候选，因此后续工作都围绕候选过滤展开。

## 2. 面积与形状规则

早期过滤从参考图拟合小球面积主峰，并加入圆度、长宽比、extent、solidity、convexity 和 circle IoU。优点是无需标注且解释直接；主要失败原因是球团、阴影球、边界截断真值与背景碎片在面积/形状空间中存在重叠。继续叠加固定阈值不能稳定解决跨倍率问题。

## 3. 种子启发式

种子方法不再只问“mask 像不像圆”，而是从原图独立检测局部球心证据，再要求 SAM 候选得到种子支持。径向对比、百分位阈值、NMS 和径向平衡明显改变了判定依据，并通过 tile 边界邻近支持减少碎片漏检。

负面结论是：局部对比阈值仍会随图像亮度、背景密度和球形态变化；阴影球、粘连球和弱对比球可能没有稳定种子，高分无种子回退又会重新引入伪候选。

## 4. Learned seed 与弧段/形状组合

随后尝试从干净参考图的正样本学习局部径向特征分布，而不是完全手工指定种子模板。增强版本把 learned seed 与面积范围、边界弧段、单球/双球几何和救回规则组合。

这条路线证明参考样本能够提供比固定阈值更贴近数据的局部证据，但系统仍由许多阶段阈值组成，参数耦合较强。它因此成为从启发式走向候选级分类器的过渡方法，而不是被删除的“失败版本”。

## 5. CNN v3 原型分类器

v3 将原图 crop、mask 形状和 masked crop 组成三通道样本，以人工标签训练轻量 CNN，并结合类别原型进行候选分类。它能够显式学习 `real_ball`、阴影球、双球团和干扰等类别，减少规则层层叠加。

尺度归一化、多尺度输入等实验显示，单纯改变 crop 尺度并不必然提升结果。历史实验中多尺度版本在部分验证集上约为 0.79 accuracy，但对阴影类别仍不足；主要限制是标注量、类别不平衡和背景域差异。

## 6. CNN v4 背景一致性

v4 针对 v3 可能学习“密集背景像正类、脏背景像负类”的捷径，对 mask 外背景进行置灰、噪声和亮度扰动，并约束 clean/augmented embedding 保持一致。采用一致性学习而非 MAE，是因为任务需要背景变化下分类稳定，而不是重建背景纹理。

## 7. CNN v4.1 尺度感知分类

v4.1 是当前保留的完整实现。除图像表示外，它显式输入候选面积、bbox 尺寸、长宽比等尺度特征，并使用 2K 六分类人工标注。当前标签共 818 条：262 real ball、38 shadow ball、227 double cluster、63 shadow double cluster、39 mixed-background single ball、189 interference。

v4.1 改善了倍率信息缺失问题，但最终上限仍受 SAM 候选召回、标注覆盖和跨域数据规模约束。仓库因此同时保留前序路线，方便定位问题究竟发生在候选生成、局部证据还是学习式分类阶段。
